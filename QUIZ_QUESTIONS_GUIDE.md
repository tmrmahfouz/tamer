## ✨ أنواع الأسئلة

### 1. اختيار متعدد (Multiple Choice) 📝
```
- عدد غير محدود من الخيارات (2 كحد أدنى)
- إجابة صحيحة واحدة
- مناسب لأغلب الأسئلة
```

**مثال:**
```
السؤال: ما هي لغة البرمجة الأكثر شعبية؟
الخيارات:
  ○ Python ✓
  ○ Java
  ○ C++
  ○ JavaScript
الإجابة الصحيحة: Python
النقاط: 2
```

### 2. صح/خطأ (True/False) ✓✗
```
- خياران فقط: صح أو خطأ
- سريع وبسيط
- مناسب للحقائق والمفاهيم
```

**مثال:**
```
السؤال: Python هي لغة برمجة مفسرة
الخيارات: صح / خطأ
الإجابة الصحيحة: صح
النقاط: 1
```

### 3. إجابة قصيرة (Short Answer) ✍️
```
- إجابة نصية حرة
- يتطلب مراجعة يدوية
- مناسب للأسئلة المفتوحة
```

**مثال:**
```
السؤال: ما هو الفرق بين List و Tuple في Python؟
الإجابة الصحيحة: List قابل للتعديل بينما Tuple غير قابل للتعديل
النقاط: 3
```

## 📝 خطوات إضافة سؤال

### 1. افتح صفحة تفاصيل الاختبار:
```
/dashboard/admin/quizzes/[quiz_id]
```

### 2. اضغط "إضافة سؤال"

### 3. املأ بيانات السؤال:

#### الحقول الأساسية:
- **نص السؤال** ⭐ (مطلوب)
- **نوع السؤال** ⭐ (مطلوب)
- **النقاط** ⭐ (مطلوب)

#### حسب نوع السؤال:

##### اختيار متعدد:
```
1. أدخل الخيارات (2 كحد أدنى)
2. اضغط "+ إضافة خيار" لخيارات إضافية
3. اختر الإجابة الصحيحة من القائمة
4. أضف توضيح (اختياري)
```

##### صح/خطأ:
```
1. اختر الإجابة الصحيحة (صح أو خطأ)
2. أضف توضيح (اختياري)
```

##### إجابة قصيرة:
```
1. اكتب الإجابة الصحيحة المتوقعة
2. أضف توضيح (اختياري)
```

### 4. احفظ السؤال:
- اضغط "إضافة السؤال"
- سيظهر السؤال في القائمة فوراً

## 🎯 أمثلة عملية

### مثال 1: سؤال اختيار متعدد

```
نص السؤال:
"ما هي البنية الأساسية لحلقة for في Python؟"

نوع السؤال: اختيار متعدد

الخيارات:
1. for i in range(10):
2. for (i=0; i<10; i++)
3. foreach i in 10
4. loop i to 10

الإجابة الصحيحة: for i in range(10):

النقاط: 2

التوضيح:
"في Python نستخدم for مع in و range() للحلقات"
```

### مثال 2: سؤال صح/خطأ

```
نص السؤال:
"يمكن تغيير قيم Tuple بعد إنشائه في Python"

نوع السؤال: صح/خطأ

الإجابة الصحيحة: خطأ

النقاط: 1

التوضيح:
"Tuple هو نوع بيانات غير قابل للتعديل (immutable) في Python"
```

### مثال 3: إجابة قصيرة

```
نص السؤال:
"اكتب دالة Python لحساب مربع عدد"

نوع السؤال: إجابة قصيرة

الإجابة الصحيحة:
"def square(x): return x ** 2"

النقاط: 3

التوضيح:
"يمكن استخدام ** للرفع لأس معين، أو x*x"
```

## 📊 نظام النقاط

### توزيع النقاط الموصى به:

#### حسب الصعوبة:
```
سهل:      1 نقطة
متوسط:    2-3 نقاط
صعب:      4-5 نقاط
```

#### حسب النوع:
```
صح/خطأ:          1 نقطة
اختيار متعدد:    2 نقطة
إجابة قصيرة:     3-5 نقاط
```

### حساب درجة النجاح:
```
إجمالي النقاط = مجموع نقاط جميع الأسئلة
درجة النجاح = (إجمالي النقاط × نسبة النجاح) / 100

مثال:
- 10 أسئلة × 2 نقطة = 20 نقطة
- نسبة النجاح 70%
- درجة النجاح = 20 × 0.7 = 14 نقطة
```

## 🔄 سير العمل الموصى به

### لإنشاء اختبار كامل:

#### المرحلة 1: التخطيط
```
1. حدد موضوع الاختبار
2. حدد عدد الأسئلة المطلوبة
3. وزع الأسئلة حسب النوع والصعوبة
4. حدد إجمالي النقاط
```

#### المرحلة 2: إنشاء الاختبار
```
1. أنشئ الاختبار من /dashboard/admin/quizzes
2. حدد الدورة والدرس
3. حدد الإعدادات (وقت، نجاح، محاولات)
4. احفظ كمسودة
```

#### المرحلة 3: إضافة الأسئلة
```
1. افتح تفاصيل الاختبار
2. أضف الأسئلة واحداً تلو الآخر
3. نوّع بين الأنواع المختلفة
4. راجع كل سؤال قبل الحفظ
```

#### المرحلة 4: المراجعة
```
1. راجع جميع الأسئلة
2. تأكد من الإجابات الصحيحة
3. راجع التوضيحات
4. تأكد من توزيع النقاط
```

#### المرحلة 5: النشر
```
1. عدّل الاختبار
2. فعّل "نشر الاختبار"
3. احفظ التغييرات
4. الاختبار متاح للطلاب
```

## 💡 نصائح مهمة

### 1. كتابة الأسئلة:
- ✅ اجعل السؤال واضحاً ومباشراً
- ✅ تجنب الأسئلة الملتبسة
- ✅ استخدم لغة بسيطة
- ✅ تأكد من وجود إجابة واحدة صحيحة

### 2. الخيارات (للاختيار المتعدد):
- ✅ اجعل جميع الخيارات معقولة
- ✅ تجنب خيارات "كل ما سبق" أو "لا شيء مما سبق"
- ✅ اجعل الخيارات متساوية في الطول تقريباً
- ✅ رتب الخيارات منطقياً (أبجدياً أو رقمياً)

### 3. التوضيحات:
- ✅ أضف توضيح لكل سؤال
- ✅ اشرح لماذا الإجابة صحيحة
- ✅ أضف معلومات إضافية مفيدة
- ✅ اجعل التوضيح تعليمياً

### 4. توزيع النقاط:
- ✅ وزع النقاط حسب الصعوبة
- ✅ اجعل إجمالي النقاط رقماً مناسباً (10، 20، 50، 100)
- ✅ تأكد من أن درجة النجاح منطقية
- ✅ راجع النقاط بعد إضافة جميع الأسئلة

## 🛠️ الـ APIs المستخدمة

### 1. جلب اختبار واحد:
```typescript
GET /api/quizzes/[id]

Response (للمدرس/أدمن):
{
  success: true,
  quiz: {
    _id: "...",
    title: "...",
    questions: [
      {
        question: "...",
        type: "multiple-choice",
        options: [...],
        correctAnswer: "...",
        points: 2,
        explanation: "..."
      }
    ]
  }
}

Response (للطالب):
{
  success: true,
  quiz: {
    _id: "...",
    title: "...",
    questions: [
      {
        question: "...",
        type: "multiple-choice",
        options: [...],
        points: 2
        // لا توجد correctAnswer أو explanation
      }
    ]
  }
}
```

### 2. تحديث الاختبار (إضافة/تعديل أسئلة):
```typescript
PUT /api/quizzes/[id]

Body:
{
  questions: [
    {
      question: "...",
      type: "multiple-choice",
      options: ["...", "...", "..."],
      correctAnswer: "...",
      points: 2,
      explanation: "..."
    }
  ]
}

Response:
{
  success: true,
  message: "تم تحديث الاختبار بنجاح",
  quiz: {...}
}
```
